# 🎓 Личный кабинет заочного студента — v1.2

**Полнофункциональная система управления учебными заданиями** для студентов и преподавателей.  
Позволяет:
- 📤 Студентам отправлять задания (с поддержкой нескольких файлов),
- 👨‍🏫 Преподавателям проверять, выставлять статусы и писать рецензии,
- 📊 Отслеживать успеваемость с датой и временем,
- 🔗 Быстро переходить на расписание, материалы и консультации.

🔗 **Демо (Render):** [https://student-cabinet-mvp.onrender.com](https://student-cabinet-mvp.onrender.com)

## 🔐 Аутентификация

Для входа требуется:
- **Студентам**: номер зачётки + пароль
- **Преподавателям**: ID преподавателя + пароль

### Для студента
- ✅ Многофайловая загрузка заданий
- ✅ Статусы: «Отправлено» → «На рассмотрении» → «Зачтено» / «Не зачтено»
- ✅ Отображение рецензии при «Не зачтено»
- ✅ Таблица успеваемости с датой и временем
- ✅ Быстрые ссылки: расписание, материалы, консультации
- ✅ Файлы редактированной работы

### Для преподавателя
- ✅ Просмотр всех отправленных работ
- ✅ Скачивание файлов с сохранением (не открытием в браузере!)
- ✅ Управление статусами: «Принят на рассмотрение», «Зачтено», «Не зачтено»
- ✅ Написание рецензий
- ✅ Автоматическая очистка файлов при «Не зачтено»
- ✅ Отправка файлов студенту

> 💡 Обе роли работают независимо друг от друга — это **единая система с разными представлениями**.

## 🗃️ Архитектура данных

### База данных
Реляционная модель с соблюдением целостности и уникальности:

- `students` — студенты с ФИО и уникальным `student_id`  
- `teachers` — преподаватели с ФИО и `teacher_id`  
- `subjects` — учебные дисциплины  
- `subject_teachers` — связь многие-ко-многим (кто ведёт предмет)  
- `assignments` — задания с дедлайнами  
- `submissions` — отправленные работы (с привязкой к файлу)  
- `grades` — итоговая успеваемость по предмету  

✅ Все связи через `FOREIGN KEY`  
✅ Ограничения: `UNIQUE`, `NOT NULL`, `CHECK`  
✅ Внешние ключи включены явно (`PRAGMA foreign_keys = ON`)

### Файловое хранилище
Файлы **не хранятся в БД** — только пути. Структура:

Структура:  
`storage/submissions/{student_id}/{assignment_id}/{timestamp}_{filename}`  


Пример:  
`storage/submissions/1/2/20260101_200000_lab1.py`

✅ Безопасные имена (фильтрация спецсимволов)  
✅ Изоляция по студентам и заданиям
